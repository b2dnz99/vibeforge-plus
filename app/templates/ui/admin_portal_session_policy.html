Session policy

Six operator-tunable knobs governing session + token lifetime. Changes take effect on next session creation / next heartbeat / next gate firing — no restart needed.
// SESSION LIFECYCLE
Heartbeat extends expires_at by this many hours on each tab-active ping. Default 24h.
Hard ceiling from created_at regardless of slide, in days. Default 7d. Constraint: slide ≤ cap.
SA + SU expire together at this many minutes. Default 15m. One knob — both cookies share the value (die-together rule per §3.1).
Active sessions per (user, type). Oldest auto-revoked when exceeded. Default 5.
// AGENT TOKENS
New tokens get expires_at = now + N days. Default 90d (matches Let's Encrypt). Eternal tokens are an explicit per-token toggle — see Agents workspace.
Sessions inactive for this many days get auto-revoked on next workspace render. Default 30d. Set 0 to disable. Tokens are NOT subject to this sweep (per §3.5).
// FORENSIC RETENTION
Soft-delete via revoked_at column — revoked rows preserved indefinitely for IP / user-agent / actor forensic data. No automatic deletion in v1; a one-line cron purges >180d-revoked rows when the table grows uncomfortable (years out at this scale).

// PROJECT WIZARD
After onboard completes, how long the wizard surfaces stay live before retiring (drawer Resume Onboarding link, /ui/onboard/{slug} resume render). Default 24 h. Same window applies whether the close-ceremony has stamped or not — both retire after this duration. Live: next render reads from SystemSetting.
